# Antigravity IDE: Harness 사용 설명서

이 프로젝트(`atworks-test`)는 일반적인 AI 코딩 어시스턴트 환경을 넘어, **에이전트 팀 아키텍처 공장(Harness)** 이 내장된 특별한 워크스페이스입니다.

Claude Code 전용이었던 Harness 플러그인의 철학과 구조를 Antigravity IDE의 Planning Mode와 태스크 오케스트레이션에 완벽하게 맞추어 이식했습니다.

## 📁 디렉토리 구조

이 하네스 시스템은 아래의 두 가지 핵심 요소로 동작합니다.

1. **`ANTIGRAVITY_HARNESS.md`** (루트 폴더)
   - 오리지널 하네스의 `SKILL.md`와 같은 **메인 트리거** 역할을 합니다.
   - AI가 하네스 명령을 받았을 때 가장 먼저 읽고 워크플로우를 기획하는 기준 문서입니다.

2. **`.gemini/skills/harness/references/`**
   - 오리지널 하네스의 `references/` 폴더 역할을 합니다.
   - 파이프라인(Pipeline), 팬아웃(Fan-out) 등의 아키텍처 패턴 가이드와 오케스트레이터 작성법 등 총 6개의 Antigravity 전용 지식 파일이 들어있습니다.

---

## 🚀 사용 방법 (프롬프트 트리거)

단순한 코드 수정이 아니라 **복잡한 시스템, 새로운 도메인 기능, 또는 풀스택 연동**이 필요할 때 AI(Gemini)에게 아래와 같이 요청하세요.

**추천 프롬프트 예시:**
> *"새로운 로그인/결제 시스템을 만들 건데, **하네스 패턴**을 적용해서 기획해 줘."*
> *"프론트엔드 성능 최적화 작업을 **하네스 방식**으로 설계하고 진행해 줘."*
> *"데이터 크롤링 및 분석 파이프라인을 **하네스로 구성**해 줘."*

### AI의 동작 방식
위와 같이 요청하면 AI는 일반적인 단발성 코딩을 멈추고 **오케스트레이터(팀 리더)** 모드로 전환하여 다음과 같이 행동합니다:

1. **지식 로딩**: `.gemini/skills/harness/references/`의 가이드들을 자체적으로 스캔합니다.
2. **아키텍처 설계**: 해당 작업에 적합한 패턴(순차적 파이프라인, 병렬 팬아웃 등)을 결정하여 우측에 `implementation_plan.md`를 기획하고 사용자님의 승인을 요청합니다.
3. **태스크 쪼개기 및 페르소나 전환**: 승인이 떨어지면 `task.md`를 작성하고, 자신을 여러 명의 개발자/QA로 분할하여(페르소나 전환) 작업을 순차적, 혹은 백그라운드 병렬로 수행합니다.
4. **검증 및 보고**: 모든 작업이 끝나면 서브에이전트(브라우저 봇 등)를 통해 QA를 진행하고 `walkthrough.md`로 최종 결과물을 시각적으로 보고합니다.

---

## 💡 언제 사용하면 좋나요?

- 프론트엔드와 백엔드를 동시에 구축해야 할 때
- 코드 작성 후 철저한 QA(교차 검증)가 필수적일 때
- 여러 API 연동이나 병렬 조사가 필요한 복잡한 기획 작업
- AI가 코드 전체 구조를 체계적으로 잡고 단계별로 진행하길 원할 때

이제 이 프로젝트에서는 AI를 단순한 코드 생성기가 아닌 **"팀 아키텍트이자 오케스트레이터"**로 활용해 보세요!
