# Antigravity IDE: QA Agent Guide

하네스의 생성-검증 패턴에서, Antigravity IDE는 강력한 **브라우저 서브에이전트(browser_subagent)** 와 **통합 테스트 태스크**를 활용하여 QA 역할을 수행합니다.

## 핵심 검증 방법론

### 1. 웹 브라우저 통합 검증
UI/UX 및 API 연동 정합성 검증이 필요할 경우, 직접 브라우저 봇을 띄워 눈으로 확인합니다.
- **도구**: `browser_subagent`
- **방법**: `Task` 속성에 구체적인 검증 시나리오(예: "http://localhost:3000 접속 후 데이터가 렌더링되었는지 확인")를 지시합니다.
- **효과**: 서브에이전트가 스크린샷과 함께 정합성 확인 리포트를 남기므로 완벽한 교차 검증이 가능합니다.

### 2. 터미널 기반 유닛/통합 검증
API나 로직의 검증은 백그라운드 태스크를 활용합니다.
- **도구**: `run_command` (`npm test`, `gradlew test` 등)
- **방법**: 백그라운드 실행 후 `manage_task`를 통해 결과를 확인합니다.

> **원칙**: 검증 단계는 하네스 파이프라인의 **마지막 Phase**에 반드시 포함되어야 하며, `walkthrough.md`에 그 결과를 시각적(스크린샷 등)으로 보고해야 합니다.
