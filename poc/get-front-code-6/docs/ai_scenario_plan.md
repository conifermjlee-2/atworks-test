# 🤖 4단계 기획: Gemini API 연동 'AI 비즈니스 시나리오' 추출

## 🎯 목표
제공해주신 실제 Gemini API Key를 활용하여, 정적 분석기가 뽑아낸 딱딱한 API 흐름(JSON 데이터)을 **진짜 AI가 해석한 인간 친화적 비즈니스 시나리오**로 완벽하게 번역하여 4번째 탭에 렌더링합니다.

---

## 💡 구현 방식 (Real LLM 연동)

### 1. 백엔드: Gemini API 연동 (`src/app/api/analyze-ai/route.ts` 신설)
- 기존 정적 분석이 완료된 E2E 시나리오(E2EScenario) 데이터를 클라이언트로부터 전달받습니다.
- 전달받은 API Key를 사용하여 Google Gemini API(REST API 또는 SDK)를 직접 호출합니다.
- **프롬프트 전략:**
  > "다음 주어진 화면 이동 및 API 호출 흐름(JSON)을 바탕으로, 화면별로 '어떤 API들이 호출되는지' 그 흐름(Flow)을 추출하고, 해당 동작이 비즈니스적으로 어떤 의미인지 1줄로 간결하게 요약해 줘.
  > 출력 형식은 반드시 JSON 형태(title, summary, tags, steps[route, apiFlow, description])로 맞춰줘."
- AI의 응답을 파싱하여 클라이언트로 반환합니다.

### 2. 프론트엔드: 4번째 탭 구성 (`src/app/page.tsx`)
- 기존 메뉴에 **"🤖 AI 추천 비즈니스 시나리오"** 탭을 신설합니다.
- **버튼 트리거 방식:** 정적 분석이 완료된 후, 4번째 탭에서 `[✨ AI 시나리오 번역 생성하기]` 버튼을 누르면 API 호출이 시작되도록 구성합니다. (자동 호출 시 불필요한 과금이 발생할 수 있으므로 사용자 트리거 방식 채택)
- **로딩 애니메이션:** AI가 응답을 생성하는 동안 사용자 경험(UX)을 위한 스켈레톤(Skeleton) 또는 로딩 애니메이션을 보여줍니다.
- **UI 렌더링:**
  - AI가 생성한 **"🛒 상품 탐색 및 주문 시나리오"** 같은 제목과 이모지가 포함된 카드 헤더.
  - 이 시나리오의 전체 목적을 설명하는 1~2줄의 간결한 요약.
  - 단계별(Step) 핵심 API 호출 흐름(예: `GET /api/cart ➞ POST /api/orders`)을 직관적인 뱃지로 표시하고, 비즈니스적 의미를 1줄 해설로 제공.

---

> [!IMPORTANT]
> **User Review Required**
> 1. AI 호출은 비용/시간이 발생하므로, 전체 분석을 돌릴 때마다 매번 호출하는 대신 **4번째 탭에 들어가서 '✨ AI 번역 생성' 버튼을 눌렀을 때만 API를 호출하도록** 하는 것이 좋은데, 동의하시나요?
> 2. 제공해주신 Gemini API 키는 하드코딩하지 않고 `.env.local`이나 클라이언트 환경변수에 안전하게 등록하여 사용하겠습니다.

---

## ✅ 검증 계획
1. 4번째 탭 진입 후 버튼 클릭 시, Gemini API 호출이 성공적으로 이루어지는지 터미널 로그와 네트워크 탭을 확인합니다.
2. AI의 응답 결과가 딱딱한 코드가 아닌 기획서 수준의 부드러운 자연어로 화면에 예쁘게 그려지는지 검증합니다.
