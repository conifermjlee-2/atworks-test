# 🏗️ [설계] 프론트엔드 코드 정적 분석기 (get-front-code)

## 기술 스택
- Next.js (App Router)
- React
- Tailwind CSS v4
- Lucide-React

## 핵심 엔진 (정적 분석)
- **glob:** `node_modules`, `dist` 등 분석 대상에서 제외하고 `.js`, `.jsx`, `.ts`, `.tsx` 파일만 수집
- **AST(TypeScript Compiler API / 추상 구문 트리):** 컴포넌트, Import, Hook, 함수 호출 등을 단순 텍스트가 아닌 **'문법 구조(트리)'**로 완벽하게 분해하여 분석합니다.
  - *도입 이유:* 단순 텍스트(정규식)나 AI 문맥 추론에 의존하면 주석 처리된 코드나 유사 변수명으로 인해 치명적인 매핑 오작동이 발생합니다. AST를 통해 코드를 100% 수학적이고 구조적으로 스캐닝하여 오차를 원천 차단합니다.
  - *엔진 신뢰성:* VS Code가 코드 에러를 잡거나 ESLint가 문법을 검사할 때 사용하는 마이크로소프트의 **공식 TS 컴파일러 엔진(`typescript` 패키지)**을 분석의 바탕에 깔고 있습니다. 즉, 컴파일러가 코드를 이해하는 것과 완전히 동일하고 완벽한 수준으로 프론트엔드 의존성을 파악해 냅니다.
- **정규식(Regex):** Axios, Fetch 등 API 호출 패턴 탐지
- **의존성 분석(Call Graph):** API 호출 이후 발생하는 화면 갱신 및 상태 변경 흐름 분석
- **simple-git:** 타겟 GitHub 저장소를 로컬 임시 디렉터리로 Clone

- AI 기반 문맥 분석 시 Hallucination으로 인해 실제 사용하지 않는 API까지 연관 관계로 분석되는 문제 확인
- 동일 코드에서도 분석 결과의 일관성이 떨어져 API 영향도 분석 및 시나리오 생성의 신뢰성이 낮음
- AST와 정규식 기반의 100% 정적 분석으로 변경하여 실제 소스 코드 기준으로만 결과를 생성하도록 개선

### ⚠️ 예시 비교 (AI 한계점)
**실제**
- A 화면 → `b-1` 기능만 사용

**AI 분석 결과**
- A 화면 → `b-1`, `b-2`, `b-3` 모두 사용하는 것으로 잘못 분석 (할루시네이션 및 과적합)

**정적 분석 결과**
- A 화면 → `b-1`만 정확하게 분석

# ⚡ 현재 구조 및 핵심 기능 (get-front-code)
설계 개편을 통해, **AI 의존 없이 순수 정적 스캐닝만으로 단 1초 만에** 타겟 프로젝트(로컬/GitHub)의 API 연계 구조와 상태 흐름을 파악하는 강력한 분석 엔진을 완성했습니다.

## 🌟 4대 핵심 기능
1. **🔌 View-API 매핑:** UI 컴포넌트와 API 간의 1:1 의존성 도출 (레거시/리팩토링 파악 시간 단축)
2. **🔄 API 연계 흐름 (Cross-Screen Flow):** 단순한 단발성 API 호출을 넘어, 특정 API(예: POST/DELETE) 호출 직후 연쇄적으로 발생하는 **데이터 갱신 흐름**을 추적합니다.

3. **💡 시나리오 추천 (Unit Action Scenario):** 특정 화면 내에서 사용자가 취하는 **단위 액션(Action)이 어떤 API 연쇄 결과(Result)로 이어지는지**에 집중한 단위 시나리오를 도출합니다.
   - **기능 단위의 인과관계 도출:** 예를 들어 `[버튼 클릭/POST] 데이터 생성` -> `[자동 GET] 목록 갱신` -> `[자동 GET] 상세 정보 갱신`처럼, 하나의 행동이 유발하는 정확한 시스템적 결과들을 묶어냅니다.
   - **효과:** 기획자와 QA 팀이 특정 기능에 대해 테스트 케이스를 작성할 때, "A 버튼을 누르면 B와 C 데이터가 갱신되어야 한다"는 **명확한 검증 기준(Test Oracle)**을 자동으로 제공받을 수 있습니다.
   
4. **📦 상태 관리 흐름:** 전역 상태(Zustand 등)의 변경 및 구독 컴포넌트 추적 (의도치 않은 상태 오염 버그 예방)
