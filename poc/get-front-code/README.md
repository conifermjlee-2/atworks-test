# ⚡ 프론트엔드 코드 정적 분석기 (get-front-code)

AI 의존 없이, 순수 정적 코드 스캐닝만으로 1초 이내에 프론트엔드 프로젝트(RTK Query 기반)의 API 연계 구조와 상태 흐름을 분석해 내는 강력한 분석 도구입니다. 

로컬 폴더를 지정하거나 GitHub URL을 입력하여 대상 프로젝트를 분석하고, 화면과 API가 어떻게 얽혀있는지 직관적인 표와 시퀀스로 뽑아줍니다.

---

## 🌟 핵심 기능 (Analysis Modes)

### 🔌 1. View-API 매핑
* **기능 설명:** 각 UI 컴포넌트 파일이 어떤 API를 호출하는지 1:1로 추출하여 매핑합니다.
* **🎯 작업 목적:** 프론트엔드 화면과 백엔드 API 간의 의존성을 직관적으로 파악하고, 특정 화면에서 어떤 데이터를 읽고 쓰는지 식별하기 위함입니다. (레거시 파악 및 리팩토링 시 화면-API 관계 파악 시간 단축)

### 🔄 2. API 연계 흐름 (Cross-Screen Flow)
* **기능 설명:** API 호출 후 RTK Query의 캐시 무효화(`invalidatesTags`)나 명시적 재호출(`refetch`)로 인해 자동으로 이어지는 연쇄 호출(단발성 호출 제외)을 추적합니다.
* **🎯 작업 목적:** 단일 동작이 화면 전체 데이터에 미치는 파급 효과(Side-effect)를 파악하여, 데이터 동기화 누락이나 불필요한 중복 갱신을 방지하기 위함입니다. (무한 렌더링 방지 및 프론트엔드 성능 최적화)

### 💡 3. 시나리오 추천
* **기능 설명:** 추출된 API 연계 흐름을 바탕으로 사용자의 행동(Action)이 어떤 결과(Result)를 낳는지 E2E 시나리오 형태로 제공합니다.
* **🎯 작업 목적:** 단순한 API 목록을 넘어 실제 비즈니스 로직 기반의 통합 테스트 시나리오를 100% 정적 분석으로 자동 도출하여, 기획자/QA의 테스트 케이스 작성 및 유지보수 비용을 획기적으로 줄이기 위함입니다.

### 📦 4. 상태 관리 흐름 (State Flow)
* **기능 설명:** Zustand, Redux 등 프론트엔드 전역 상태(State)가 어느 컴포넌트에서 읽히고 변경되는지 추적합니다.
* **🎯 작업 목적:** 복잡한 화면 간 상태 공유 구조를 한눈에 파악하고, 의도치 않은 전역 상태 오염으로 인한 버그를 추적 및 예방하기 위함입니다.

## 🚀 시작하기

### 개발 서버 실행

```bash
npm run dev
# 또는
yarn dev
# 또는
pnpm dev
```

브라우저에서 [http://localhost:3000](http://localhost:3000) 로 접속하여 결과를 확인합니다.

### CLI 분석 스크립트 실행
터미널 환경에서 UI 없이 빠르게 결과물만 추출(마크다운 형태)하고 싶다면 아래 스크립트를 실행합니다.

```bash
npx tsx run_analyzer.ts
```
*(분석 결과는 `api-flow-result.txt` 등의 파일에 저장됩니다.)*
